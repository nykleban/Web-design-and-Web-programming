Звіт лабораторна робота 1 - Vue 3, TypeScript та Vue Router

1) Деталізовані кроки виконання

    1. Створення та ініціалізація проєкту
    - Використано команду npm create vue@latest для створення нового проєкту Vue.
    - Обрано опції TypeScript та Vue Router для налаштування проєкту.

    2. Встановлення та запуск
    - Перехід до директорії проєкту та інсталяція залежностей командою npm install.
    - Запуск проєкту для первинної перевірки справності.

    3. Налаштування маршрутизації (Vue Router)
    - Створено новий компонент-сторінку ContactView.vue у директорії src/views.
    - У файлі src/router/index.ts прописано новий маршрут /contact, що вказує на ContactView.
    - Додано елемент <RouterLink to="/contact"> до навігаційної панелі для доступу до нової сторінки.

    4. Створення та типізація компонента (TypeScript)
    - Створено компонент UserCard.vue у директорії src/components.
    - Пропси компонента типізовані за допомогою TypeScript (defineProps), зокрема пропс isActive визначено як boolean.

    5. Інтеграція та перевірка
    - У ContactView.vue імпортовано та використано компонент UserCard, передаючи необхідні пропси (username, isActive).
    - Проєкт запущено для фінальної перевірки відображення двох карток на сторінці "/contact".

2) Результат тестування помилок (TypeScript)

    При спробі передати значення типу number замість очікуваного boolean для пропсу isActive 
    (наприклад, <UserCard isActive="1" />), компілятор TypeScript запобігає цій дії та видає наступну помилку

    Текст помилки
    Type 'number' is not assignable to type 'boolean'.
    (property) isActive boolean

    Висновок про помилку
    Ця помилка підтверджує, що TypeScript успішно застосовує статичну типізацію, 
    виявляючи невідповідність типів ще на етапі розробки, що є ключовою перевагою TS.

3) Висновки про використані технології

    TypeScript додає статичну типізацію до JavaScript. Це допомагає виявляти помилки на етапі розробки 
    (до запуску коду), покращує підтримку коду (особливо у великих проєктах) та робить його самодокументованим 
    (легше зрозуміти, які дані очікує функція чи компонент).

    Vue Router забезпечує створення SPA (Single Page Application), дозволяючи користувачам 
    переходити між сторінками (компонентами) без повного перезавантаження браузера, що значно 
    покращує користувацький досвід та швидкість роботи. Також підтримує ліниве завантаження для 
    оптимізації початкового завантаження.
